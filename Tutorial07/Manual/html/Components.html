コンポーネント一覧

Transform

機能

全てのオブジェクトが持っているコンポーネント。

ゲームオブジェクトの位置、向き、大きさの情報を持っています。

ゲームオブジェクトの親子関係の情報も持っています。

Inspectorパラメーター

Position : ローカルの座標。

Rotate : ローカルの回転。2πで１回転。

Sacle : ローカルの大きさ。

Script変数 Script関数 注意

現在インスペクター上で削除する事が出来るが、実際には削除されていない。（してはならない。）

PhysXコンポーネントがあるとRotateのパラメーターがクォータニオンになる。さらにwを弄れないのでエディター上では操作不能。

Model

機能

モデルファイルのデータを持つコンポーネント。

描画をするにはMeshDraw、Materialのコンポーネントが必要。

Inspectorパラメーター

Model : モデルのパスを指定してファイルを読み込む。

　　　　例[ Assets/__.tesmesh ]

　　　　対応ファイルはtesmesh,tedmesh

Bone : ボーンファイルのパスを指定してファイルを読み込む。

　　　　例[ Assets/__.tebone ]

　　　　対応ファイルはtebone

Script変数 Script関数 注意

アニメ無しはtesmeshのみ読み込む。

アニメ有りはtedmeshとteboneを読み込む。さらにアニメーションコンポーネントとvmdの読み込みもする。

アニメ有りの場合はメッシュ変更やボーン変更を行っても正常に出来ない可能性が高い。

IKの計算が不完全。

SDEFのウェイトの計算に未対応。

TextureModel

機能

画像やテキストの表示用のメッシュを持つコンポーネント。

画像を描画をするにはMeshDraw、Materialのコンポーネントが必要。

テキストを描画するにはTextコンポーネントが必要。

Inspectorパラメーター Script変数 Script関数 注意

画像描画ではMaterialでTextureとTexture.fxの読み込みも必要。

このコンポーネントを入れるとTransformコンポーネントのポジションなどのパラメータが独自の指定になるがGUIは変更されないので現在描画位置設定は困難。

Material

機能

モデルの描画に必要なマテリアルの設定をするコンポーネント。

Inspectorパラメーター

Material : マテリアルのパスを指定してファイルを読み込む。

　　　　例[ Assets/__.txt ]

　　　　対応ファイルはpmxの変換によって出力されたtxt

r,g,b : マテリアルの色。

Texture : テクスチャのパスを指定してファイルを読み込む。

　　　　例[ Assets/__.png ]

　　　　対応ファイルはpng,jpg,bmpなど

Shader : シェーダーのパスを指定してファイルを読み込む。

　　　　例[ Assets/__.fx ]

　　　　対応ファイルはfx,hlsl

Script変数 Script関数 注意

現在デファードシェーディングに描画を変えた関係でマテリアルのRGB設定が反映されていない。

テクスチャーを指定してもマテリアルの０番にしか反映されない。

Text

機能

テキストの文章を持つコンポーネント。

テキストを描画するにはTextureModelコンポーネントが必要。

Inspectorパラメーター Script変数 Script関数 注意

ゲームプレイ中でないと表示が見えない。

Animation

機能

モデルのアニメーションファイルを読み込みと再生を行うコンポーネント。

Inspectorパラメーター

Time : アニメーションの再生位置。

VMD : アニメーションファイルのパスを指定してファイルを読み込む。

　　　　例[ Assets/__.vmd ]

　　　　対応ファイルはvmd

Script変数 Script関数 注意

モデルでボーンなどを読み込んでいると一度ゲームプレイをしないと表示されない。

ゲームプレイ中勝手に再生されてループなどの処理もない。

Camera

機能

ゲームプレイ中のカメラを設定するコンポーネント。

Inspectorパラメーター Script変数 Script関数 注意

現在ゲームシーンに複数このコンポーネントが存在しても片方が無視される。

Script

機能

スクリプトで作成したクラスを導入するコンポーネント。

またスクリプトのリフレクションで指定した変数をInspector上に表示する。

Inspectorパラメーター

Class : スクリプトで作成したクラス名を入力すると反映される。

Script変数 Script関数 注意

ゲームプレイとストップ切替時にInspectorを更新しないと独自に作成したクラスの変数の入力フォームのポインターが古い。

PhysX

機能

オブジェクトに重力を与える。

当たり判定の処理などを行う。

PhysXColliderコンポーネントが自分自身または子に存在しないと誰とも当たらない。

Inspectorパラメーター

Kinematic : 有効にすると重力の影響を受けなくなり、別のオブジェクトに当たっても押されなくなる。

Script変数 Script関数 注意

PhysXCollider

機能

オブジェクトに当たり判定を与える。

PhysXコンポーネントが自分自身または親に存在しないと誰とも当たらない。

Inspectorパラメーター

Mesh : モデルのパスを指定するとメッシュ形状の当たり判定を作成する。

有効になるとBox,Sphereの当たり判定は無効化される。

　　　　例[ Assets/__.tesmesh ]

　　　　対応ファイルはtesmesh

IsSphere : 無効の場合、Box形状の当たり判定が作成される。

有効になるとSphere形状の当たり判定が作成される。

IsTrigger : 有効になるとこのオブジェクトとの当たり判定が消える。

ヒット時にスクリプトではヒットコールバック関数が呼ばれる。

Script変数 Script関数 注意

PostEffect

機能

ポストエフェクトの描画を追加する。

Inspectorパラメーター Script変数 Script関数 注意

現在ポストエフェクトの変更なども行えない。

PointLight

機能

デファードシェーディングのポイントライトを追加します。

Inspectorパラメーター

radius : ポイントライトの影響範囲。

AttenuationStart : ポイントライトの計算パラメーター。

AttenuationParam : ポイントライトの計算パラメーター。

r,g,b : ポイントライトの色。

Script変数 Script関数 注意

Particle

機能

GPUで計算したパーティクルを表示する。

Inspectorパラメーター Script変数 Script関数 注意

現在描画周りの変更などにより利用できない。
